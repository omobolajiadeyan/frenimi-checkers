# Contributing

Contributions are welcome when they improve correctness, security,
accessibility, portability, or test coverage.

1. Open or reference an issue.
2. Keep changes focused.
3. Add regression tests.
4. Run `npm run check`, `npm test`, and `npm run audit`.
5. Do not commit secrets, databases, player data, binaries, or live malicious
   payloads.

All pull requests require review and successful CI checks.
